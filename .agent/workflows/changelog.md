# CHANGELOG 更新ルール（絶対ルール）

このプロジェクト（`studioours24_form`）に **何か変更を加えたら、必ず `CHANGELOG.md` を更新すること。**

## 更新タイミング
- コードの変更をコミットする **前** に `CHANGELOG.md` を更新する
- 1つのコミットに含まれる変更はまとめて1エントリにする

## フォーマット

```markdown
## YYYY-MM-DD

### [絵文字] 変更タイトル
- `変更したファイル名`: 変更内容の簡潔な説明
```

### 絵文字ガイド
- 🎉 新機能 / 初回リリース
- ✏️ テキスト・コンテンツ変更
- 💰 料金・金額関連
- 🔧 バグ修正 / 動作改善
- 🎨 デザイン・スタイル変更
- 📄 ページ追加
- 🗑️ 削除
- ⚙️ 設定・インフラ変更

## 注意事項
- 新しい変更は **ファイルの上部（日付セクション直下）** に追記する
- 同じ日に複数の変更がある場合は同じ日付セクション内に追記する
- `CHANGELOG.md` の更新も変更と一緒にコミットする
